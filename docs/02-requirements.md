# 需求规格说明书

## 1. 项目概述

### 1.1 项目名称

WearableAgent Hub

### 1.2 项目目标

在 PC 本地构建一个穿戴设备 AI Agent 原型平台，完整验证以下能力：

1. A2A 协议的多 Agent 通信能力
2. A2UI 协议的结构化穿戴 UI 渲染能力
3. x402 协议的服务计费与结算能力
4. 真实模型接入能力（当前默认 MIMO OpenAI 兼容接口）

### 1.3 目标用户

- 学习和验证 A2A/A2UI/x402 协议的开发者
- 验证穿戴 AI 场景的产品/方案人员
- 需要可运行原型做演示或研究的人

### 1.4 项目范围

#### 包含

- 眼镜 HUD 模拟器
- 手表表盘模拟器
- Agent 管理面板
- 核心 Agent 后端
- A2A Server/Client 能力
- A2UI 穿戴渲染能力
- x402 支付模拟能力
- 示例 Agent（翻译、导航、支付）
- 语音交互模拟能力

#### 不包含

- 真实穿戴硬件接入
- 主网真实支付
- 生产级部署架构
- 移动端适配

## 2. 核心需求

### 2.1 本地可运行

系统必须在单台 PC 上可完整运行，包含：

- 核心后端
- 前端模拟器
- 示例 Agent
- 支付服务
- 管理面板

### 2.2 协议链路完整

系统需要覆盖三条主链路：

#### 链路 A：输入 → LLM → A2UI 渲染

用户输入文字/语音模拟信号 → Agent 调用模型 → 生成结构化结果 → 前端 HUD/表盘渲染。

#### 链路 B：主 Agent → 子 Agent

主 Agent 识别任务意图后，通过 A2A 协议调用远程 Agent，并拿到结果。

#### 链路 C：付费服务调用与结算

当任务需要付费服务时，系统通过 x402 完成服务请求、结算、状态返回。

### 2.3 真实模型接入

当前阶段需要具备真实 LLM 接入能力，默认使用：

- `OPENAI_BASE_URL=https://token-plan-cn.xiaomimimo.com/v1`
- `OPENAI_MODEL=mimo-v2.5-pro`
- `PROVIDER_TYPE=openai_compatible`

这表示：

- 配置层已经接入真实 MIMO API
- 调用层使用标准 OpenAI chat-completion 格式
- 项目默认不以 mock 作为主路径

### 2.4 穿戴 UI 优先

UI 需求不是“做一个漂亮后台”，而是：

- 适配小屏信息展示
- 强调状态、标题、摘要、列表等轻量化信息
- 支持 HUD 风格消息卡片与状态栏
- 支持手表表盘类数据展示

### 2.5 可观测

系统需要提供基本可观测能力：

- Agent 列表
- 通信日志
- Task 状态
- 支付记录
- 当前后端配置信息

## 3. 功能需求

### 3.1 核心后端

核心后端需要提供：

- 健康检查接口
- 聊天接口
- WebSocket 会话接口
- A2A Server 能力
- A2UI 消息生成能力
- 语音会话入口
- 配置化模型接入能力

### 3.2 眼镜模拟器

眼镜模拟器需要提供：

- 文字输入
- HUD 消息展示
- A2UI 组件渲染
- 状态反馈
- 支付状态展示
- 语音入口（可为模拟模式）

### 3.3 手表模拟器

手表模拟器需要提供：

- 表盘类信息面板
- 迷你输入
- 迷你消息展示
- 与后端实时通信能力

### 3.4 Dashboard

Dashboard 需要提供：

- Agent 状态总览
- Payments 页面
- Agents 页面
- Logs 页面

### 3.5 示例 Agent

至少提供三类示例 Agent：

- 翻译 Agent
- 导航 Agent
- 支付 Agent

用于验证：

- A2A 基本通信
- 工具型服务调用
- 付费服务调用

## 4. 非功能需求

### 4.1 可复现

项目应在以下环境稳定运行：

- Windows 10+
- Python 3.13（conda `expr`）
- Node.js 20+
- pnpm 9.x

### 4.2 可扩展

后续应能较容易扩展：

- 新的 Agent
- 新的 A2UI 组件
- 新的支付场景
- 新的语音链路

### 4.3 可维护

关键逻辑应分层清晰：

- 配置层
- 引擎层
- 协议层
- UI 层
- 示例层

### 4.4 可演示

项目需要适合演示：

- 能快速启动
- 能看到完整链路
- 能清楚解释每个模块作用
