# 设计目标与决策

## 1. 项目愿景

WearableAgent Hub 的目标不是做一个完整的穿戴硬件产品，而是构建一个**可在 PC 本地运行的穿戴 AI Agent 原型平台**。

它要回答三个核心问题：

1. 穿戴场景下的 Agent 通信是否适合用 A2A
2. 穿戴场景下的信息展示是否适合用 A2UI
3. 穿戴场景下的 API 付费服务是否适合用 x402

因此，这个项目的本质是：

- 一个协议联调平台
- 一个穿戴 AI 交互原型
- 一个可演示、可学习、可扩展的实验系统

## 2. 设计目标

### 目标 1：真实链路可跑通

项目必须能完整跑通以下链路：

#### 链路 A：输入 → 推理 → A2A → A2UI → HUD

用户输入 → AgentEngine 调用 LLM → 意图识别 → 可选调用远程 Agent → 生成结构化结果 → A2UI 渲染 → 前端穿戴界面展示。

#### 链路 B：Agent → Agent → x402 → 结算

用户请求触发付费服务 → 主 Agent 识别支付意图 → 调用 x402 服务完成结算 → 返回结果并展示支付状态。

### 目标 2：真实模型可接入

项目应默认具备接入真实模型的能力，而不是只依赖 mock。

当前默认模型接入方式为：

- MIMO OpenAI 兼容接口
- `OPENAI_API_KEY`
- `OPENAI_BASE_URL=https://token-plan-cn.xiaomimimo.com/v1`
- `OPENAI_MODEL=mimo-v2.5-pro`
- `PROVIDER_TYPE=openai_compatible`

并且代码中已经通过标准 `chat.completions.create` 进行调用。

这意味着：

- MIMO 已经作为配置层真实接入
- 当前项目已经具备真实 LLM 调用能力
- 但完整链路还需要在 A2A / A2UI / x402 场景下进行端到端验证

### 目标 3：穿戴优先

UI 不是做通用管理后台，而是做穿戴设备场景：

- AI 眼镜 HUD
- AI 手表表盘
- 小屏信息密度优化
- 状态栏、卡片、列表、消息气泡等轻量化展示

### 目标 4：协议优先

这个项目的重点不是前端炫技，而是协议验证：

- A2A 验证多 Agent 发现与协作
- A2UI 验证结构化 UI 渲染
- x402 验证服务计费与结算

### 目标 5：本地可运行

整个系统必须能在一台 PC 上完成：

- 启动
- 演示
- 调试
- 排错

不依赖硬件，不依赖复杂部署。

### 目标 6：可讲解、可交接

项目需要具备清晰的文档与结构，能够支撑：

- 自己复盘
- 他人理解
- 方案汇报
- 后续继续迭代

## 3. 设计原则

### 原则 1：先真实，再美化

优先保证链路真实跑通，再优化 UI、交互、错误提示。

### 原则 2：先配置驱动，再代码硬编码

所有关键参数都应来自配置文件，尤其是：

- 模型 API
- base_url
- 模型名称
- provider 类型
- 服务端口

### 原则 3：先单机联调，再分布式扩展

当前阶段统一在本地运行，后续再考虑：

- 多机部署
- 远程 Agent 注册
- 云端服务接入

### 原则 4：先最短链路，再复杂场景

优先保证以下链路稳定：

1. 用户输入 → LLM 回复 → HUD 渲染
2. 主 Agent → 子 Agent → 结果返回
3. 付费服务调用 → x402 结算 → 状态展示

## 4. 当前关键技术决策

### 决策 1：后端使用 Python + FastAPI

因为：

- a2a-sdk 官方 Python 版本最成熟
- FastAPI 原生支持 WebSocket 与异步接口
- 适合 Agent 编排与本地原型开发

### 决策 2：LLM 使用 OpenAI 兼容接口

因为：

- 统一接口标准
- 可快速接入不同兼容模型
- 当前默认接入 MIMO

### 决策 3：A2UI 自研穿戴渲染器

因为：

- 官方渲染器偏向通用 Web
- 穿戴场景需要更紧凑的信息展示
- 自研更容易适配 HUD 与表盘风格

### 决策 4：x402 使用独立微服务桥接

因为：

- x402 官方 SDK 更偏 TypeScript 生态
- Python 主服务通过 HTTP 调用支付服务更简单
- 本地演示场景下复杂度可控

## 5. 成功标准

项目成功不是“文档写得好”，而是满足以下标准：

### 标准 1：最短链路可演示
用户输入后，能在穿戴模拟器上看到结果。

### 标准 2：A2A 真实可调用
主 Agent 能成功调用子 Agent 并返回结果。

### 标准 3：x402 真实可结算
付费服务调用后能完成模拟支付闭环。

### 标准 4：MIMO 接入真实可验证
后端能基于真实 API 配置完成推理调用。

### 标准 5：仓库结构可理解
别人看仓库能快速知道：
- 目标是什么
- 架构是什么
- 怎么启动
- 怎么验证
